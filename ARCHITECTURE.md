# ARCHITECTURE.md — Homelab Bot

## Суть проекта

Telegram-бот на Rust, живущий в приватном чате с единственным разрешённым пользователем. Использует Forum Topics (Threaded Mode) вместо отдельного webapp — вся функциональность идёт через топики внутри одного чата с ботом.

## Конфигурация

Все секреты и идентификаторы хранятся в `.env`:

- `BOT_TOKEN` — токен бота
- `CHAT_ID` — ID приватного чата
- `NOTIFICATIONS_THREAD_ID` — thread_id топика "Уведомления" (заполняется автоматически при первом запуске)
- `QUICK_NOTES_THREAD_ID` — thread_id топика "Быстрые заметки" (заполняется автоматически при первом запуске)
- `NOTIFY_SERVER_PORT` — порт HTTP-сервера для приёма внешних нотификаций

При старте бот проверяет `NOTIFICATIONS_THREAD_ID` и `QUICK_NOTES_THREAD_ID`. Если оба пусты — создаёт топики через Bot API и записывает полученные `message_thread_id` обратно в `.env`.

## Архитектура запуска

Два независимых tokio-задачи в одном процессе:

```
main.rs
├── task: Bot (teloxide dispatcher)
│   └── слушает входящие в "Быстрых заметках"
│
└── task: HTTP-сервер (axum)
    └── ловит нотификации с локалки → шлёт в "Уведомления"
```

Общее состояние — `Arc`:
- `Arc<Bot>` — оба используют для отправки сообщений
- `Arc<Config>` — оба читают `thread_id`

Запуск через `tokio::spawn` + `tokio::select!` с логированием падения. Без retry/restart — `Restart=on-failure` в systemd поднимет весь процесс.

## Модули

### 1. Быстрые заметки

Топик "Быстрые заметки". Пользователь пишет текстовые сообщения прямо в этот топик, бот слушает входящие сообщения, отфильтрованные по `message_thread_id`, и сохраняет их.

### 2. Уведомления

Топик "Уведомления". Только вывод от бота в чат, пользователь туда не пишет.

Источник — независимый существующий Python-скрипт бэкапа (не трогаем). Канал доставки — внутренний HTTP на порту `NOTIFY_SERVER_PORT`.
